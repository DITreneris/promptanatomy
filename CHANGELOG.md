# Changelog

## [Unreleased] / 2026-03-08

### Prideta
- **Vercel–Supabase integracija:** Backend aplinkoje (Vercel arba lokaliai) sukonfigūruoti `SUPABASE_URL` ir `SUPABASE_SERVICE_ROLE_KEY`; Supabase lentelė `user_access` naudojama GET /api/access ir webhook upsert. Stripe webhook – `STRIPE_WEBHOOK_SECRET` įrašytas į .env; po apmokėjimo (`checkout.session.completed`) backend atnaujina prieigą Supabase.
- **Mobilus UI / UX / vartotojo kelionė (LT–EN) auditas:** Dokumentas `docs/audit-mobile-ux-user-journey.md` – mobilus UI (viewport, touch targets, breakpoints, mobile menu), UX (CTA, klaidos, loading, focus trap siūlymas, kontrastas), vartotojo kelionė ir LT/EN nuoseklumas (LocaleContext, vertimai, success/cancel). Įtraukta į `docs/INDEX.md`.
- **MVP upgrade (Supabase + access):** Backend – `GET /api/access?email=` (highest_plan, allowed_modules, can_upgrade_to); Supabase modulis `backend/db.py` (`get_user_access`, `upsert_user_access`); config – `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `plan_id_to_value`, `PLAN_VALUES`. `create-checkout-session` – prieš checkout tikrinama prieiga pagal email; jei jau turi tą patį/aukštesnį planą → 409; Stripe Session su `metadata.plan` ir `client_reference_id`. Webhook `checkout.session.completed` – iš sesijos imamas email ir `metadata.plan`, upsert į `user_access` (new_highest = max(current, purchased)), saugomas `stripe_customer_id`. Frontend – `getAccess(email)` api.js; HomePage – blokas „Patikrinti prieigą“ (email + mygtukas), state `access`/`customerEmail`; Pricing – rodo tik `can_upgrade_to` planus, „Jau turite“ / „Upgrade į X“, perduoda email į checkout. Vertimai: checkAccess, emailPlaceholder, yourAccess, alreadyHave, upgradeTo, errors.accessNotConfigured, errors.alreadyPurchased. Dokumentacija: `docs/supabase-user-access.sql` (lentelė user_access), README/TODO/.env.example atnaujinti. Testai: TestGetAccess (400, 503, 200), test_already_purchased_returns_409.
- **Deploy (produkcija):** Repozitorija nustumta į GitHub ([DITreneris/promptanatomy](https://github.com/DITreneris/promptanatomy)); frontend deploy į Vercel su `vercel.json` (install/build iš `frontend/`, output `frontend/dist`); domenas promptanatomy.app prijungtas per Porkbun DNS (A + CNAME į Vercel). Aplikacija veikia adresu promptanatomy.app / www.promptanatomy.app.
- **Deploy / Webhook:** Dokumentuota, kad Stripe webhook reikalauja viešo backend URL; be deploy (pvz. Vercel arba kito serverio) webhook eventai neturi kur būti siunčiami, todėl `STRIPE_WEBHOOK_SECRET` (whsec_) realiai nebus naudojamas – žr. TODO.
- **Kainodara (4 lygiai):** Backend – `CreateCheckoutBody.plan_id` ("1"|"2"|"3"|"4"), config `stripe_price_id_plan_1` … `_4`, `create-checkout-session` naudoja atitinkamą Stripe Price; frontend – 4 planų kortos (39 / 99 / 149 / 199 €) `Pricing.jsx`, `createCheckoutSession(planId)` per `api.js`; refund tekstas LP pakeistas į „Grąžinimo sąlygos – susisiekite“. Testai: `test_missing_plan_id_returns_422`, `test_invalid_plan_id_returns_422`, `test_plan_not_configured_returns_400`; `test_checkout_not_configured_returns_503` atnaujintas.
